Herramientas Utilizadas

VS Code

Editor para HTML, CSS y JS.

Git Bash

Terminal para comandos Git.

GitHub

Gestión de repositorios.

Gemini AI

Asistente de codificación.

GitHub Pages

Hosting del sitio web.

Proceso de Desarrollo

Cronología de versiones y evolución del código

Configuración Inicial

Fase 1: Preparación del Entorno

Creación del directorio de trabajo y vinculación con el sistema de control de versiones.

  • Creación de carpeta 'Pagina-web'
  • Comando: git init
  • Configuración de identidad Git
Estructura Semántica

Fase 2: Versión 1 (Básica)

Desarrollo del archivo index.html inicial con asistencia de Gemini AI para definir la jerarquía de datos.

  • Estructura HTML única
  • git add index.html
  • Primer commit 'primera version'
Colaboración y Organización

Fase 3: Versión 2 (Intermedia)

Refactorización del código dividiéndolo en carpetas lógicas para mayor escalabilidad.

  • Directorios: assets y css
  • Clonación del repositorio: git clone
  • Nuevos commits por integrantes
Optimización y Lanzamiento

Fase 4: Versión Final y Despliegue

Pulido visual, inclusión de logos, perfiles completos y activación de GitHub Pages.

  • Sección de Proyectos integrada
  • Autorización de credenciales
  • Hosting en GitHub Pages activo

Conclusión del Proyecto

La combinación de herramientas como VS Code, Git Bash y GitHub permitió un control total del ciclo de vida del software. El uso de Gemini AI aceleró el prototipado, permitiendo al equipo enfocarse en la calidad del diseño final.

Logros Clave

Dominio de sistemas de control de versiones descentralizados y despliegue continuo (CI/CD) básico.

Sugerencias

Uso de ramas (branches) secundarias para pruebas experimentales y mayor granularidad en los commits.
